## Setup

### 1. Clone the repo
```bash
git clone <your-repo-url>
cd repo-2-rfm
```

### 2. Place dataset files
Place all 7 CSV files inside the `data/` folder:

```
data/
├── customers.csv
├── orders.csv
├── support_tickets.csv
├── web_events_snapshot.csv
├── churn_labels.csv
├── rfm_modeling_snapshot.csv
└── intervention_history.csv
```

> The dataset is provided separately as part of the capstone brief. Do not commit CSVs to this repo.

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Run the notebook
```bash
jupyter notebook rfm_segmentation.ipynb
```

Run all cells top to bottom (`Kernel → Restart & Run All`).  
`segments.csv` and `segment_summary.csv` are written to the working directory.  
Charts are saved as `rfm_chart_*.png`.

## Segmentation Logic

| Segment | Rule | Churn Rate |
|---|---|---|
| Champions | recency ≤ 30d, frequency ≥ 5, monetary ≥ ₹4,000 | 7.7% |
| Loyal but at risk | frequency ≥ 4, recency > 90d | 89.2% |
| Support-stressed | ticket_count ≥ 1, negative sentiment > 50% | 42.9% |
| Discount hunters | avg_discount ≥ 30% | 53.1% |
| Dormant | recency > 180d, sessions = 0 | 84.9% |
| Maintain | All others | 48.8% |

Non-RFM signals used: support ticket count, negative sentiment rate, sessions (30d), abandoned carts.

## Budget Recommendation (₹25,000)

| Priority | Segment | Rationale |
|---|---|---|
| 1 | Loyal but at risk | Highest revenue at risk — proven buyers now going cold |
| 2 | Support-stressed | Churn driven by service friction, fixable |
| 3 | Dormant | Low-cost reactivation email on remaining budget |
| — | Champions | Loyalty recognition only, no spend needed |
| — | Discount hunters | Margin-safe bundle offers, not blanket discounts |
